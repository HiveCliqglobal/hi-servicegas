Why permanent · Pre-flight

The temp token dies in 24 hours

Meta's developer dashboard issues a temporary token labelled "24 hours" on the WhatsApp API Setup page. In practice that token starts returning HTTP 401 the moment its 24-hour window closes — and every outbound message stops dead until it's replaced. Production WhatsApp ordering cannot run on temp tokens. The fix is one System User token, generated once, that never expires.

"Error validating access token: Session has expired on Friday, 22-May-26 00:00:00 PDT. The current time is Friday, 22-May-26 00:14:01 PDT."

The business.facebook.com dropdown shows a business that owns the WhatsApp app. Already set up in this case — Hi-Service Orders app was created here.

2 · "Hi-Service Orders" app with WhatsApp product added

The app exists at developer.facebook.com under the Hi-Service business. Use case "Connect with customers through WhatsApp" already selected. Already set up.

3 · A WABA visible in the WhatsApp app's API Setup page

"Test WhatsApp Business Account" (id 1357146642926345) provisioned by Meta. The sandbox phone +1 555-629-8543 is its From-number. Already verified working.

State: ready to generate the permanent token

Why both?

WABA grants the token permission to act on WhatsApp
App grants the token an identity (the App ID it acts as)
One without the other → 401 errors

Step 5 · Copy with Meta's button, not the keyboard

Click Meta's Copy button inside the dialog. Do NOT triple-click + Cmd-C — keyboard selection sometimes captures an invisible line break that Meta's API will silently reject. Token starts with EAA… and is ~200–300 characters.

Step 6a · Verify the paste before saving

Open TextEdit / Sticky / a Gmail draft → paste → confirm: one unbroken string, starts with EAA, no spaces, no line breaks. If anything looks off, generate a new token (no penalty).

Critical: closing Meta's dialog without copying loses the token forever

Step 6b · Paste into Hi-Service admin

Open hiservice.store/admin/connections.php → Meta WhatsApp card → click Update Meta credentials → System User Token field → Cmd+A → Delete → Paste the new token → Save Meta. The field is pre-filled with the OLD dead token's masked dots — overwriting is essential.

Step 7 · Troubleshooting · Verification

If something goes wrong

Once saved, ping the build session with "saved" — a 4-endpoint Graph API probe runs immediately to verify the token works before any real message is sent. If a snag does come up earlier, the table below maps the symptom to the exact step that fixes it.

SnagSeverityWhat it meansFix
Generate Token button stays greyed out High Asset assignment didn't complete — token cannot be generated against zero assets. Re-do Step 3A and 3B. The button activates only after both assets save.
Only whatsapp_business_management visible, no _messaging High WABA was assigned but without full control — Meta only grants management-level scopes. Step 3A again. Toggle Full control ON (or both "Manage WhatsApp Business" and "Manage phone numbers and messages").
App dropdown empty in the Generate dialog High The app asset was never assigned to the system user. Step 3B. Add Hi-Service Orders as an Apps asset with Full control.
Token saved but still 401 on test Med Either a line break got into the paste, or the field still holds the old dead token's value. Step 6b again. Cmd+A to clear first, then paste. Verify prefix differs from EAAbZBpyiPS3UBRpy…
"You don't have permission" page Med You're not Admin on the Business Portfolio that owns the app. Business Settings → Business Info → check who has Admin. Ask owner to elevate, or use their account.

Standing by for "saved"

Once Step 6b is done, ping the active build session with the word "saved" and the 4-endpoint probe runs immediately. If all 4 return HTTP 200, you reply "hi" to +1 555-629-8543 and the echo arrives within five seconds — closing the inbound to outbound WhatsApp loop.
